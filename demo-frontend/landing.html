The Problem

Back tables are sterile workspaces used in operating rooms to organize surgical instruments prior to and during a procedure. Each surgery demands a highly specific and standardized layout, yet mistakes in back table setup remain common.

A study in BMJ Quality & Safety (2019) found that setup errors account for up to 23% of intraoperative delays, directly impacting patient safety and surgical outcomes. Manual validation is time-consuming, dependent on experience, and often undocumented.

The Solution

A computer vision-powered, head-mounted real-time back table setup validator that compares visual inputs with procedure-specific layouts and instrument lists in seconds â€” not minutes.

How It Works

1. Procedure Selection

Clinician or scrub tech selects the planned surgical procedure (e.g., laparoscopic cholecystectomy).

2. Layout Generation

Our system scrapes clinical resources and OR prep protocols to generate a tailored list of required tools and a visual layout for the back table.

3. Real-Time Scan

Using a wearable camera (e.g., mounted on glasses or forehead), the scrub nurse scans the setup.

4. Instrument Identification

Segmentation models isolate each item on the table, which are then sent to the Google Lens API for identification.

5. Cross-Referencing

The system cross-references detected tools against the required list and ideal layout.

6. Instant Feedback

Real-time feedback is displayed: missing instruments, extra items, or incorrect placements.

Final Pipeline

Goal: Seamlessly validate surgical back table setups to ensure procedure readiness and sterility

User Interaction Flow:

  • User inputs planned surgery or procedure.
  • System scrapes online protocols and databases (e.g., AORN, OR guidelines, textbooks) to generate a visual and itemized back table layout.
  • User performs scan of back table using a head-mounted camera.
  • Video is segmented frame-by-frame to isolate individual tools and containers.
  • Each segmented image is sent through the Google Lens API to identify the instrument.
  • Items are grouped, counted, and listed.
  • List is compared against required setup (including optional instruments).
  • Dashboard shows visual checklist, alerts missing or extra tools, and saves reports for documentation.

Semantic Image Search w/ OpenAI CLIP + Embedding Search:

  • Input: Folder of segmented, cropped instrument images
  • Output: Identified object name + quantity
  • Post-processing: Aggregate duplicate detections, sort into categories (e.g., clamps, retractors)

MCP Server (Agentic) for Scraping:

  • Input: Selected surgical procedure
  • Action: Use agentic automation to pull data from surgical prep guidelines, institutional protocols, and trusted medical repositories
  • Output:
    • Instrument list (by type and brand)
    • Layout guides or reference images (if available)
    • Notes on optional or surgeon-specific preferences
